# Quick Access Guide for Microsoft Enterprise Users

**Having trouble accessing this repository?** Follow these steps:

## 1. Enable Two-Factor Authentication (2FA)
- Go to [GitHub Settings > Security](https://github.com/settings/security)
- Enable 2FA using Microsoft Authenticator or another method
- **This is required for most Microsoft enterprise policies**

## 2. Link Your Microsoft Email
- Go to [GitHub Settings > Emails](https://github.com/settings/emails)
- Add and verify your Microsoft enterprise email (@microsoft.com)
- You can keep your personal email as primary

## 3. Request Access
Contact the repository owner (@msftsean) with:
- Your GitHub username
- Your Microsoft enterprise email
- Why you need access

**Methods to request**:
- Open a GitHub issue with label "access-request"
- Send a direct message on Teams
- Email with your GitHub username

## 4. Authorize SSO (if prompted)
- When cloning the repository, you may see an SSO authorization prompt
- Click "Authorize" and complete the Microsoft login
- For SSH keys: Go to Settings > SSH Keys and click "Enable SSO" on each key

## Common Issues

### "Repository not found"
→ You need access from the repository owner

### "Permission denied" when cloning
→ Check if 2FA is enabled and if you need to authorize SSO

### Can't push changes
→ You may need Write access (request from owner)
